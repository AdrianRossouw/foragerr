# foragerr — Security Risk Register (FRG-PROC-006)

One row per risk, current state only. **L** / **I** are likelihood / impact on an `L`/`M`/`H`
scale.

**Status** is one of:
- **Open** — no control in place yet; mitigation planned or deferred.
- **Mitigated** — a control now protects against the risk (including "impossible/immune by
  construction").
- **Accepted** — owner-accepted residual; carries a review trigger.
- **Closed** — no longer applicable.
- **Withdrawn** — no longer considered a risk.

**Current mitigation** states, in present tense, what protects against the risk now and cites the
governing requirement(s) — `FRG-...` IDs and the draft requirement headings they resolve to
(`IF`/`PF`/`LD`/`ACQ`/`FD`/`SEC-new`). Accepted rows — and mitigated rows carrying a live residual —
state their review trigger.

Historical state changes live in version control — `git log --follow docs/security/risk-register.md`
reconstructs any risk's journey; rows are replaced, never appended to.

**Source** names the finding each risk came from: a research finding (`opds S1`, `ddl §4`, `cv §4`,
`fs §N`, `sonarr §N`, `filename-parsing §4`) or `analysis`, plus the change/proposal or threat-model
node that governs the row's current state.
