# foragerr — Security Risk Register (FRG-PROC-006)

Read-only analysis staged outside the repo per FRG-PROC-008. One row per risk. `L/M/H` =
likelihood / impact. **Decision**: *mitigate* cites the governing draft requirement (by its
`### AREA — <name>` heading, which the completeness-critic audit confirmed resolve 1:1 to the
now-allocated FRG IDs) or a NEW requirement in
`sec-nfr-requirements.md` (prefixed `SEC-new:`); *accept* gives rationale + a review trigger.
"Source": research finding (`opds S1`, `ddl §4`, `cv §4`, `fs §N`) or `analysis`.

Every security flag in the research docs is represented (traceability column notes which).
Draft files: LD=library-domain, ACQ=acquisition, FD=files-domain, IF=interfaces, PF=platform.
